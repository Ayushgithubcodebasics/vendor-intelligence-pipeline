PROJECT NOTES
=============

This project analyzes purchasing, sales, inventory, and vendor invoice data for
a beverage alcohol distributor — 15.6 million rows across six CSV files covering
the 2024 fiscal year.

Why I built it:
  I wanted a project that involved genuinely messy data, not a pre-cleaned toy
  dataset. The vendor name problem alone — same supplier registered under multiple
  legal-name variants across the three source files — took me most of a weekend
  to handle properly. I also wanted something I could walk through in an interview
  without having to explain away a contrived example.

  The ExciseTax field in sales.csv made the industry obvious from the start.
  Alcohol excise is collected from the distributor and passed through, so it
  doesn't affect gross profit but it does inflate the sales price for non-analyst
  readers. I kept that in mind when setting up the margin calculations.

What the pipeline does:
  Reads all six source CSVs in memory-safe chunks, builds a canonical vendor name
  map by taking the most frequently seen name variant per vendor ID, computes
  weighted average purchase price across chunks (tracking running dollar and
  quantity totals rather than averaging chunk means), allocates freight cost to
  vendor-brand rows by purchase-dollar share, and calculates lead time and OTIF
  from purchase order dates. The final output is one summary file per analytical
  dimension plus a validation report that reconciles back to raw totals.

Tests:
  pytest tests/ — 14 tests, all pass on the full dataset.
  One known data quality issue: all 126 measurable vendors fall into
  TIER_1_RELIABLE on OTIF, so that column has no differentiation value on this
  particular dataset. The test suite flags this as a warning rather than a failure.

Known issues I would fix with more time:
  - No CI/CD pipeline yet (GitHub Actions would run pytest on sample data per push)
  - The run script is Windows-only (PowerShell); no bash equivalent for Mac/Linux
  - No unit test specifically for the weighted average price logic across chunks
  - OTIF SLA is a 14-day proxy; a tighter window might produce tier separation
